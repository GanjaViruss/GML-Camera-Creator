Cameras in GameMaker trip a lot of people up — views, viewports, the application surface, the GUI layer, and aspect ratio all interact, and it's easy to end up with a blurry or stretched mess. This tool generates a clean, complete camera object from a few choices, so you skip the boilerplate and the head-scratching.

What you get

One script with camera_init(), camera_update() and optional helpers like camera_shake() and camera_set_zoom(). You paste it into a script asset, then call the functions from an obj_camera. The tool shows you exactly which call goes in which event.

How to set it up
  1. Make a new Script asset (e.g. scr_camera) and paste the generated script into it.
  2. Make an object obj_camera, place it in your room (no sprite needed).
  3. Put the calls in the events the tool tells you: Create → camera_init(), Step → camera_update() (plus input if you enabled zoom).
  4. Set your room's viewport/camera to match your base resolution, or just let the code handle it. Run the game.

obj_player is just a placeholder for whatever the camera follows — use your own object. The generated code is plain GML; open it and tweak anything. To trigger a shake from anywhere (e.g. on hit): with (obj_camera) camera_shake(4, 20);

About GUI scaling (the "Scaled" option)

The GUI (HUD) layer can work three ways. Match base and Custom let GameMaker stretch a fixed GUI size to the screen — simple, but on a pixel-art game the HUD gets blurry when stretched. Scaled fixes that: it sets the GUI to the real screen size and gives you a GUI_SCALE value. You multiply your HUD draws by it, so everything stays crisp and the right size on any screen.

It works out to 1 at your design size, 2 at double, 0.5 at half, and so on. Use it like this in Draw GUI:

draw_sprite_ext(spr_heart, 0, x, y, GUI_SCALE, GUI_SCALE, 0, c_white, 1);
draw_text_transformed(x, y, "SCORE", GUI_SCALE, GUI_SCALE, 0);

On top of that there's a user multiplier — global.gui_scale_user (1.0 = normal). Call gui_scale_increase() / gui_scale_decrease() from your options menu and the whole HUD scales up or down by your chosen step (default 5%). So someone on a 4K screen who still wants a bigger HUD can just bump it. Save global.gui_scale_user with your settings to remember it.
